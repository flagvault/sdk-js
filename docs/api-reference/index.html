API Reference

This page documents all the classes, methods, and interfaces available in the FlagVault SDKs.

FlagVaultSDK​

The main class for interacting with the FlagVault service.

Constructor​

Creates a new instance of the FlagVault SDK.

  • JavaScript/TypeScript
  • Python
constructor(config: FlagVaultSDKConfig)

Parameters​

NameTypeDescription
configFlagVaultSDKConfigConfiguration options for the SDK

Throws​

  • Error if apiKey is not provided

Example​

const sdk = new FlagVaultSDK({
  apiKey: 'live_your-api-key-here', // Use 'test_' for test environment
  timeout: 10000 // Optional: timeout in milliseconds
});
__init__(api_key: str, timeout: int = 10)

Parameters​

NameTypeDescription
api_keystrAPI Key with environment prefix (live_/test_)
timeoutintOptional: Request timeout in seconds (default: 10)

Raises​

  • ValueError if api_key is not provided

Example​

# Basic usage
sdk = FlagVaultSDK(
    api_key='live_your-api-key-here'  # Use 'test_' for test environment
)

# With custom timeout
sdk = FlagVaultSDK(
    api_key='live_your-api-key-here',
    timeout=30  # Longer timeout
)

Methods​

isEnabled / is_enabled​

Checks if a feature flag is enabled.

  • JavaScript/TypeScript
  • Python
async isEnabled(flagKey: string): Promise<boolean>
Parameters​
NameTypeDescription
flagKeystringThe key for the feature flag
Returns​

A Promise that resolves to a boolean indicating if the feature is enabled.

Throws​
  • Error if flagKey is not provided
  • Error if the API request fails
Example​
try {
  const isEnabled = await sdk.isEnabled('my-feature-flag');
  console.log(`Feature is ${isEnabled ? 'enabled' : 'disabled'}`);
} catch (error) {
  console.error('Error checking feature flag:', error);
}
is_enabled(flag_key: str, default_value: bool = False) -> bool
Parameters​
NameTypeDescription
flag_keystrThe key for the feature flag
default_valueboolDefault value to return on error (default: False)
Returns​

A boolean indicating if the feature is enabled, or default_value on error.

Raises​
  • ValueError if flag_key is not provided
Example​
# Simple usage - returns False on any error
is_enabled = sdk.is_enabled('my-feature-flag')
print(f"Feature is {'enabled' if is_enabled else 'disabled'}")

# With custom default value
is_enabled = sdk.is_enabled('experimental-feature', default_value=True)

# No try/catch needed - errors are handled gracefully
# The SDK will log error messages but won't raise exceptions

Types and Interfaces​

Configuration Types​

  • JavaScript/TypeScript
  • Python

FlagVaultSDKConfig​

Configuration options for the FlagVault SDK.

interface FlagVaultSDKConfig {
  /**
   * API Key with environment prefix for authenticating with the FlagVault service.
   * Use 'live_' prefix for production, 'test_' prefix for test environment.
   * Can be obtained from your FlagVault dashboard.
   */
  apiKey: string;

  /**
   * Optional request timeout in milliseconds.
   * Defaults to 10000 (10 seconds).
   */
  timeout?: number;
}

Configuration Parameters​

The Python SDK accepts the following parameters in the constructor:

class FlagVaultSDK:
    """
    FlagVault SDK for Python
    
    Args:
        api_key (str): API Key with environment prefix for authenticating with the FlagVault service.
                      Use 'live_' prefix for production, 'test_' prefix for test environment.
                      Can be obtained from your FlagVault dashboard.
        timeout (int, optional): Request timeout in seconds.
                                 Defaults to 10.
    """

Error Handling​

Both SDKs provide detailed error messages to help you diagnose issues:

  • JavaScript/TypeScript
  • Python

Common Errors​

  • Missing Credentials: Thrown when apiKey is not provided
  • Invalid Flag Key: Thrown when flagKey is empty or not provided
  • Network Errors: Thrown when the API request fails due to network issues
  • Authentication Errors: Thrown when the provided credentials are invalid

Error Handling Example​

try {
  const sdk = new FlagVaultSDK({
    apiKey: process.env.FLAGVAULT_API_KEY, // Should be 'live_...' or 'test_...'
  });
  
  const isEnabled = await sdk.isEnabled('my-feature');
  // Use the feature flag
} catch (error) {
  if (error.message.includes('apiKey')) {
    console.error('Configuration error:', error.message);
  } else if (error.message.includes('network')) {
    console.error('Network error:', error.message);
  } else {
    console.error('Unexpected error:', error);
  }
}

Error Handling (v1.1.0+)​

The Python SDK now handles errors gracefully without raising exceptions during flag evaluation:

  • Missing Credentials: ValueError raised only during initialization
  • Invalid Flag Key: ValueError raised if flag_key is empty
  • Network Errors: Returns default_value and logs error message
  • Authentication Errors: Returns default_value and logs error message
  • Flag Not Found: Returns default_value and logs informational message

Error Handling Example​

# SDK initialization still requires error handling
try:
    sdk = FlagVaultSDK(
        api_key=os.environ.get('FLAGVAULT_API_KEY'),  # Should be 'live_...' or 'test_...'
    )
except ValueError as e:
    print(f'Configuration error: {e}')
    sys.exit(1)

# Flag evaluation no longer needs try/catch
is_enabled = sdk.is_enabled('my-feature', default_value=False)
# Errors are logged but won't crash your application

# Example log output on error:
# FlagVault: Failed to connect to API for flag 'my-feature', using default: False
